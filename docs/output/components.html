Alerts

Alert, confirm and prompt dialog boxes follow the same class pattern:

.alert
  .alert-content
    .alert-content-text
    .alert-content-btns

The only difference between the three are whether they have one or two button and the contents of .alert-content-text.

Input Groups

An input group is a combination of an input and a label. Both .text-input and .text-input-label are base styles, however when placed inside a .text-input-group their behaviour is modified.

The pop-up label effect can be achieved with pure css by adding a 'required' attribute to the corresponding input, however this could have adverse effects in some cases, so it can also be acheived by added the .up class to the label with javascript. This effect only exists when both .text-input and .text-input-label are inside a .text-input-group.

<div class="text-input-group col-6">
  <input type="text" class="text-input" required id="text-input" />
  <label class="text-input-label" for="text-input">Text Input Group Label</label>
</div>
<div class="text-input-group col-5">
  <select class="select-input" required id="text-input-2">
    <option>Select Option</option>
  </select>
  <label class="text-input-label" for="text-input-2">Select Input Group Label</label>
</div>

Sidebar

The sidebar is a view with a dark background and white text that has an .open and closed state. Most elements inside the sidebar are base styles, and are not styled inside the .sidebar component.

Click the sidebar in the example to toggle the .open state.

subhead in sidebar Icons in sidebar

  <span class="icon-with-label flex-col-1">

  </span>
